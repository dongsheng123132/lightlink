# LightLink - 手机间光通信

## 项目概述

LightLink 是一个基于 Web 的手机间光通信原型，利用手机**屏幕闪光**作为信号源、**摄像头**作为接收器，实现两台手机之间的**零网络、零蓝牙**数据传输。

### 核心理念
- **SOS 握手协议**：双方先发送 SOS (···---···) 莫尔斯信号建立"连接"
- **光通信数据传输**：握手后切换到莫尔斯编码传输文字消息
- **双方都需安装**：两台手机都打开同一个网页即可

---

## 技术架构

```
手机A (发送端)                    手机B (接收端)
┌──────────────┐                ┌──────────────┐
│  用户输入文字  │                │   前置摄像头   │
│      ↓       │                │      ↓       │
│  莫尔斯编码   │                │  亮度分析算法  │
│      ↓       │                │      ↓       │
│  屏幕闪光信号  │  ══光信号══▶  │  脉冲检测     │
│  (绿色/黑色)  │                │      ↓       │
│              │                │  莫尔斯解码   │
│              │                │      ↓       │
│              │                │  显示接收消息  │
└──────────────┘                └──────────────┘
```

### 信号协议

| 阶段 | 编码方式 | 说明 |
|------|---------|------|
| 握手 | SOS莫尔斯 | `···---···` 标准莫尔斯码 |
| 数据 | 莫尔斯码 | 支持 A-Z, 0-9, 常用标点 |
| 同步 | 前导码 | 5个快速闪烁用于时钟同步 |
| 结束 | 长闪烁 | 表示消息传输完毕 |

### 速度配置

| 模式 | 点时长 | 划时长 | 约速率 |
|------|--------|--------|--------|
| 慢速 | 200ms | 600ms | ~4 WPM |
| 中速 | 120ms | 360ms | ~7 WPM |
| 快速 | 70ms  | 210ms | ~12 WPM |

### 接收算法
- **绿色通道加权**：信号使用绿色 (#00ff88)，接收端重点分析绿色通道
- **自适应阈值**：根据近60帧亮度自动计算高低阈值
- **滞后检测**：防止噪声导致的误触发
- **自适应单位时间**：根据接收到的脉冲自动估算发送速度

## 部署方式

### 方式1：Vercel 部署 (推荐)

```bash
# 1. 初始化 git
cd ai之间的光通信
git init
git add .
git commit -m "Initial LightLink prototype"

# 2. 推送到 GitHub
gh repo create lightlink --public --source=. --push

# 3. 部署到 Vercel
npx vercel --prod
# 或者在 vercel.com 导入 GitHub 仓库
```

部署后会得到一个 HTTPS URL (如 `lightlink.vercel.app`)，两台手机打开这个 URL 即可使用。

### 方式2：GitHub Pages

```bash
# 在 GitHub 仓库 Settings → Pages → Source 选择 main 分支
# 自动部署到 https://username.github.io/lightlink/
```

### 方式3：本地测试

```bash
# 需要 HTTPS (摄像头权限要求)
# 方法A: python 简单服务器 (仅限 localhost)
python3 -m http.server 8080
# 手机访问 http://电脑IP:8080 (注: 非 localhost 需要 HTTPS)

# 方法B: 使用 mkcert 生成本地证书
mkcert -install
mkcert localhost 192.168.x.x
npx http-server -S -C localhost+1.pem -K localhost+1-key.pem -p 8080
```

**重要**：摄像头 API (`getUserMedia`) 要求 HTTPS 或 localhost。本地 HTTP 在手机上无法访问摄像头。

## 使用方法

### 基本流程
1. 两台手机打开同一个 LightLink URL
2. 将两台手机**屏幕面对面**放置 (前置摄像头对着对方屏幕)
3. 手机 A 点击 **SOS** 按钮 → 屏幕底部信号区闪烁 SOS
4. 手机 B 检测到 SOS → 自动回复 SOS
5. 双方进入"已连接"状态
6. 点击快捷消息或输入文字 → 屏幕闪烁发送 → 对方解码显示

### 设置选项
- **信号速度**: 慢速(稳定)/中速(推荐)/快速(近距离)
- **检测灵敏度**: 1-10, 越高越敏感
- **摄像头**: 前置/后置
- **信号颜色**: 绿色(推荐)/白色/青色

### 快捷消息
| 按钮 | 发送内容 | 含义 |
|------|---------|------|
| SOS | SOS莫尔斯 | 建立连接 / 求救 |
| 你好 | HI | 打招呼 |
| 收到 | OK | 确认 |
| 帮助 | HELP | 请求帮助 |
| 是 | YES | 肯定 |
| 否 | NO | 否定 |

## 扩展方向 (原型应用)

### 1. 光救援信号器
- 日间：屏幕高亮闪烁 + GPS 坐标编码
- 夜间：手电筒 API 闪烁莫尔斯码
- AI 辅助：自动生成结构化求救信息 (位置、伤情、人数)

### 2. 手机控电脑 (ESP32 USB HID)
- 手机 AI 看屏幕 → 理解界面 → 生成操作指令
- 通过蓝牙/声波发送给 ESP32-S3-GEEK
- ESP32 模拟键盘鼠标操作电脑
- 适用于：数据中心运维、内网安全终端、老旧设备

### 3. 光名片交换
- 两台手机面对面闪一下
- 交换联系方式 (vCard 编码)
- 比 NFC 更通用 (所有手机都支持)

### 4. 光密语通信
- 加密层：AES 加密后的密文通过光信号传输
- 旁人看到闪光也无法解码
- 适用于：隐私敏感场景

### 5. 光对战游戏
- 石头剪刀布：双方同时发信号
- 密码猜测：一方发数字, 另一方猜
- 反应速度测试：看到对方闪光后最快回应

### 6. 声光混合通信 (GibberLink 方向)
- 手机扬声器发超声波 (ggwave) + 屏幕光信号
- 两个通道同时传输, 互为冗余
- AI-to-AI 自主切换最高效协议

## 技术栈

- **前端**: 纯 HTML/CSS/JS, 无框架依赖
- **摄像头**: MediaDevices API (`getUserMedia`)
- **信号分析**: Canvas 2D + 像素亮度分析
- **编码**: 莫尔斯码 + 曼彻斯特编码 (计划)
- **部署**: 静态文件, Vercel/GitHub Pages

---

## 已知限制

1. **速度**: 莫尔斯码传输约 1-2 字符/秒, 不适合长文本
2. **距离**: 屏幕光信号有效距离约 30cm-1m
3. **中文**: 当前仅支持英文/数字, 中文通过快捷消息映射
4. **环境光**: 强光环境下检测准确率下降
5. **iOS Safari**: 部分设备摄像头权限可能受限
